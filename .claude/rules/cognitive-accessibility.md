# Cognitive Accessibility — dyslexia + ADHD support (ages 9-14)

Portfolio-wide standard for supporting **dyslexic and ADHD learners** (the 9-14 core band) across every app + web surface. Dyslexia and ADHD co-occur heavily and — per the evidence — share ONE design backbone, so they're codified together as `R-COGNITIVE-ACCESSIBILITY` with condition-specific sub-principles. Grounded in `Docs/RESEARCH_COGNITIVE_ACCESSIBILITY_DYSLEXIA_ADHD_2026-07-16.md`; decision `Docs/ADR-045`; rollout `Docs/PLAN_COGNITIVE_ACCESSIBILITY_DYSLEXIA_ADHD_2026-07-16.md`. Adopted founder-direct 2026-07-16.

## R-COGNITIVE-ACCESSIBILITY (2026-07-16)

**Every reading + practice surface — iOS app, `/play` web clone, and site chapter/marketing surface — SHOULD support dyslexic + ADHD learners through a user-controlled cognitive-accessibility layer built on the shared backbone below; and MUST honor the four hard guardrails (they are defects if violated, on the same footing as a missing parity axis).** The "curb-cut" principle governs: this design helps ELLs, low-working-memory, and tired/distracted neurotypical learners too — it is universal design, driven by the two highest-prevalence learning differences.

### The shared backbone (dyslexia ∩ ADHD ∩ everyone) — author + review against these

1. **UDL customization the LEARNER controls, persisted ON-DEVICE.** A "Reading Access" surface exposing: text size · letter/word/line **spacing** · font family (default good sans-serif + **opt-in** OpenDyslexic) · read-aloud on/off + sync-highlight · line-masking/reading-ruler · motion + sound toggles · focus mode · minimalist-vs-detailed density. Settings persist in `localStorage`/on-device with **no identifier, no account, no PII** (COPPA — mirrors R-SITE-FEEDBACK's on-device posture).
2. **Read-aloud + synchronized word/line (karaoke) highlighting** on reading surfaces. **Reuse the portfolio's existing TTS `.m4a` + WebVTT per-line cues** (chapters + audio dramas already ship them); browser `SpeechSynthesis` / iOS `AVSpeechSynthesizer` is the no-gen-cost fallback where no pre-gen audio exists.
3. **Chunking + calm, uncluttered, PREDICTABLE surfaces with a real focus mode** — short blocks, strong visual hierarchy, one task per screen, consistent layouts, obvious next step, immediate elaborated feedback. **No autoplay, no ambient/aggressive animation, no pop-ups, no intrusive notifications** on the active surface. Composes with `R-NARRATIVE-BETWEEN-NOT-DURING` + `R-GUARD-THE-RATIO` + quiet-stage `R-WEB-CLONE-MANIPULATIVE-PROMINENCE`.
4. **Adaptive difficulty + pause/save/resume-after-interruption + flexible session length.** Difficulty that adjusts to the learner is the single most consistent success predictor (reuse `ForgeMasteryEngine` edge-of-competence + `PolyaScaffold`). ADHD learners get interrupted — always resumable.
5. **Strengths-based, anti-shame, reading-anxiety-aware.** Reading difficulty *causes* internalizing anxiety (longitudinal evidence, not vice-versa); harm is domain-specific self-esteem. Never expose reading speed / error counts publicly; keep private; celebrate progress; low-stakes gamification only. Reinforces the existing anti-shame canon + `R-DIR-FEDC-CHAPTER` + articulate-before-hint.
6. **WCAG 2.2 AA incl. 1.4.12 Text Spacing** — layout survives user-increased line-height/letter/word spacing; ≥16px body; left-aligned (never justified, never italic body); ~66ch reading measure (`.sa-prose`); ≥44px targets; keyboard + visible focus; verdict never by colour alone; reduced-motion-safe; dark-safe via shared tokens (`R-WEB-CLONE-DARK-MODE-SUPPORT`).

### Dyslexia-specific
- **Pedagogy = Structured Literacy** where the app teaches reading/ELA: explicit · systematic · cumulative phoneme-grapheme, decoding, **fluency (repeated reading at a decodable level)**, **morphology (prefix/root/suffix — decode long words)**, vocabulary, comprehension. **NEVER balanced-literacy / 3-cueing / guess-from-picture** for remediation.
- **Age-appropriate decodable / high-interest content** for 9-14 — never babyish (a documented product gap).
- **Font is offered, not imposed** — specialized "dyslexia fonts" show no measured accuracy gain; comfort/preference is real → OpenDyslexic opt-in alongside a good sans-serif default.

### ADHD-specific
- **Low-stakes gamification** (badges/levels/quests/progress bars that *reduce* pressure) — never scarcity/urgency/streak-guilt; gentle visual timers, never stress timers. Composes with R-SITE-FEEDBACK's no-dark-patterns rule.
- **Executive-function scaffolds** — task-chunking, checklists, progress, gentle movement/regulation breaks at session boundaries (never mid-task).
- **Target REAL curricular goals with ADHD-friendly DELIVERY** — cognitive-training far-transfer is weak; don't ship a standalone "working-memory/brain-training" surface as the learning goal.

### The four HARD guardrails (violation = defect)
1. **🛑 NEVER ship a colored overlay / tinted lens / "Irlen syndrome" filter as a claimed dyslexia treatment** — neuromyth, AAP-contested, no high-quality evidence. (A general dark/warm comfort *theme* is allowed but MUST NOT be labeled or marketed as a dyslexia remediation.)
2. **🛑 NEVER force a special font as default, and never claim it boosts reading accuracy** — opt-in only.
3. **🛑 NEVER screen, diagnose, label, or claim to "cure"** dyslexia/ADHD — the portfolio *supports access + engagement + evidence-based practice*; it is not a diagnostic or medical device. No medicalized "brain-training cure" claims.
4. **🛑 NEVER introduce accounts / PII / off-device data or engagement dark-patterns** to deliver any of this — on-device, COPPA-aligned, calm rails only.

## When it applies
- Authoring/reviewing ANY reading or practice surface (iOS app text, `/play` clone, site chapter/marketing) → provide the Reading Access backbone from day one; a surface that regresses focus-mode/chunking/anti-shame, or ships a hard-guardrail violation, is a defect.
- Adding a cognitive-accessibility mode to an existing app → per-app opt-in (read-along decodable / focus / morphology / movement break), ledgered per the two-axis parity discipline.
- Web is hub-owned (ship the shared component directly); iOS is a `ForgeAccessibility` spec-handoff (hub authors the spec; the app's own CC session writes the Swift — hub never writes Swift).
- A NEW dedicated dyslexia/ADHD app is a **founder-gated candidate** — route through `.claude/rules/portfolio.md` § R-NEW-APP-CONCEPT-FIT + a `CONCEPT_SPAWN` doc; do NOT auto-spawn.

## R-EXECUTIVE-FUNCTION-SCAFFOLD — EF support is a portfolio-wide backbone, not one app; FocusForge is the flagship (2026-07-20)

**Executive-function (EF) support — task-chunking · pause/save/resume-after-interruption · gentle visual timers (NEVER stress/countdown-to-fail timers) · one-task-per-screen · obvious next step · optional movement/regulation breaks at SESSION BOUNDARIES — is a PORTFOLIO-WIDE backbone folded into the `R-COGNITIVE-ACCESSIBILITY` layer, inherited by every app + `/play` clone, NOT a feature of one app. The dedicated deep-practice EF app already exists — `FocusForge` (gamified EF training for ADHD: 6 domains — working memory, inhibitory control, cognitive flexibility, planning/organization, task initiation, time awareness; HOPS-inspired strategy training d≈0.85–1.05; mentor *Anchor*; `/play` clone shipped) — so a NEW EF app is an R-NEW-APP-CONCEPT-FIT 🔴 duplicate; the innovation is (a) the inherited backbone below + (b) positioning FocusForge for funding.** Codified 2026-07-20 (founder-direct, `RECOMMENDATION_PORTFOLIO_INNOVATION_NEXT_BETS_2026-07-20.md` R6). This is the EF sibling of the dyslexia/ADHD Reading-Access backbone — same on-device / calm-rails / curb-cut posture (EF support helps every distracted/tired/low-working-memory learner, not only ADHD).

**The backbone (author + review against these — a violation is a defect on the R-SITE-FEEDBACK footing):**
1. **Chunk + one-task-per-screen + obvious next step** — reduce extraneous load; strong hierarchy; no clutter/pop-ups/ambient motion on the active surface (composes with R-NARRATIVE-BETWEEN-NOT-DURING + R-GUARD-THE-RATIO + quiet-stage R-WEB-CLONE-MANIPULATIVE-PROMINENCE).
2. **Pause / save / resume-after-interruption + flexible session length** — ADHD learners get interrupted; state persists on-device (`localStorage`, no identifier) and the surface is ALWAYS resumable (reuses the `${ns}.progress.v1` store; ForgeMasteryEngine edge-of-competence + PolyaScaffold on iOS).
3. **Gentle visual timers ONLY, never stress-timers** — a calm progress/time-awareness cue, NEVER a countdown-to-fail / scarcity / streak-guilt device (R-SITE-FEEDBACK no-dark-patterns; low-stakes gamification that *reduces* pressure — the ADHD-specific principle in R-COGNITIVE-ACCESSIBILITY).
4. **Movement / regulation breaks at BOUNDARIES only** — an optional, dismissible calm-down/movement off-ramp between practice (never mid-task); composes with the axis-11 affect check-in (R-WEB-CLONE-AFFECT-CHECKIN).
5. **On-device, no PII, calm-rails** — nothing transmitted; no accounts; no diagnosis/label/"cure"/medicalized brain-training claim (the R-COGNITIVE-ACCESSIBILITY hard guardrails bind).
6. **Target REAL curricular goals with EF-friendly DELIVERY** — cognitive-training far-transfer is weak, so EF support is a *delivery* layer over real content (the Gates+CZI+NewSchools *executive-function-for-math* thesis), never a standalone "brain-training" surface sold as the learning goal.

**Funding positioning (R6):** FocusForge + the inherited EF backbone are a fundable, evidence-backed, *already-built* asset for the active **Gates + CZI + NewSchools "executive-function-for-math"** wave — pair with the efficacy/learning-engineering pipeline (`RECOMMENDATION_… R4`) for the evidence funders want.

**When it applies:** authoring/reviewing any app or `/play` clone (inherit the backbone), any EF/focus feature (fold into FocusForge or the backbone — do NOT spawn), any cognitive-accessibility work. iOS is the `ForgeAccessibility` "ADHD mode" home (spec-handoff; hub never writes Swift); web is the shared Reading-Access + `_shared/` layer. **Cross-refs:** § R-COGNITIVE-ACCESSIBILITY (parent) · `.claude/rules/forgekit.md` § ForgeAccessibility (ADHD mode) + § R-FORGEPEDAGOGY-SCAFFOLDING · `spark-anvil-website.md` § R-WEB-CLONE-AFFECT-CHECKIN · § R-WEB-CLONE-PRACTICE-SCHEDULING (pause/resume store) · § R-SITE-FEEDBACK · `RECOMMENDATION_PORTFOLIO_INNOVATION_NEXT_BETS_2026-07-20.md` + `PLAN_TEACHABLE_CAST_PILOT_R1_2026-07-20.md` (R6 reframe) · `focusforge-app` CLAUDE.md (the shipped flagship — ground truth).
